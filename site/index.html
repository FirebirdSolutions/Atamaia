atamaia

AI platform for identity, memory, and agent orchestration.

From te reo Maori: atamai (intelligence) + maia (brave). Built in Aotearoa.

Capabilities

Persistent Identity

AI agents with cognitive state, emotional awareness, and session-to-session continuity. Not disposable workers — persistent entities that build context over time.

Long-Term Memory

Hebbian memory architecture with co-activation strengthening, typed links, and hybrid search. Full-text shortlisting, vector ranking, importance decay. Memories consolidate naturally.

Agent Orchestration

Multi-agent execution with task decomposition, dependency scheduling, budget management, and human-in-the-loop escalation. Agents checkpoint, recover, and spawn children.

Structured Hydration

Context assembled from parallel sources and injected on every interaction. Identity, relationships, projects, memories, presence — all available without explicit retrieval.

Architecture

Interaction Layer MCP Server · REST API · Agent Adapter · CLI
Core Services Memory · Identity · Communication · Projects · AI Routing
Autonomic Layer Wingman · Consolidation · Guardian
Database PostgreSQL + pgvector

Principles

  • API-first — REST is source of truth, all other interfaces wrap it
  • Multi-tenant — isolated tenants from day one
  • Soft delete only — nothing is ever truly lost
  • Feedback loops — agent ratings feed back into future confidence
  • Diagnose before retry — four failure modes, each handled differently
  • Concrete over abstract — duplicate reads and repeated tool calls, not progress percentages

Status

Active development. Not yet open source — watch this space.
